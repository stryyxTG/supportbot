# Telegram support bot

Бот работает как тикет-система: пользователь пишет обращение прямо в Telegram, а админ обрабатывает тикеты в Telegram Mini App.

## Возможности

- создание обращения без выбора категории;
- переписка пользователя с поддержкой внутри тикета;
- поддержка текста, фото, документов, видео, голосовых и других вложений;
- несколько активных обращений на пользователя с лимитом;
- Mini App админка с разделами `Новые тикеты`, `Активные тикеты`, `Закрытые тикеты`;
- просмотр пользователя, истории сообщений и вложений в Mini App;
- ответ пользователю из Mini App;
- закрытие тикета из Mini App;
- номер тикета в формате `№1`, `№2`, `№3`;
- аудит событий в SQLite.

## Запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python bot1.py
```

## .env

```env
BOT_TOKEN=123456:replace_me
ADMIN_IDS=123456789,987654321
ADMIN_USERNAMES=admin_username,second_admin
DATABASE_PATH=data/support.sqlite3
SUPPORT_TITLE=Техническая поддержка
MAX_OPEN_TICKETS=3
WEBAPP_URL=https://your-domain.example/admin
WEBAPP_HOST=0.0.0.0
WEBAPP_PORT=8080
```

`WEBAPP_URL` должен быть публичным HTTPS URL, который открывает `/admin`. Локальный сервер поднимается на `WEBAPP_HOST:WEBAPP_PORT`; для Telegram его нужно вывести наружу через домен, reverse proxy или tunnel.

## Логика пользователя

Пользователь нажимает `Новое обращение`, пишет проблему одним сообщением и получает номер тикета. Если у него один активный тикет, следующие обычные сообщения автоматически добавляются туда. Если активных тикетов несколько, бот просит выбрать нужный.

## Логика админа

Админ пишет `/admin` и открывает Mini App.

Разделы:

- `Новые тикеты` — новые обращения и тикеты, где пользователь снова написал после ответа оператора;
- `Активные тикеты` — тикеты в работе или ожидающие ответа пользователя;
- `Закрытые тикеты` — закрытые обращения.

Внутри тикета админ видит данные пользователя, историю сообщений, вложения, может взять тикет в работу, ответить и закрыть его.

## Важно

Перед продом перевыпустите токен в BotFather, если он где-то отправлялся в чат. Для большого потока позже лучше заменить SQLite на PostgreSQL.
